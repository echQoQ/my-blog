Perfectly Secret Encryption

The Syntax of Encryption

An encryption scheme is defined by three algorithms Gen, Enc, and Dec, as well as aspecification of message space M\mathcal{M}M with ∣M∣>1|\mathcal{M}| > 1∣M∣>1

Gen

The key-generation algorithm Gen is a probabilistic algorithm that outputs a key kkk chosen according some distribution.

We denote K\mathcal{K}K the (finite) key space, i.e., the set of all possible keys that can be output by Gen.

Enc

The encryption algorithm Enc takes as input key k∈Kk \in \mathcal{K}k∈K and a message m∈Mm \in \mathcal{M}m∈M, and outputs a ciphertext ccc.

Dec

The decryption algorithm Dec takes as input a key k∈Kk \in \mathcal{K}k∈K and a ciphertext c∈Cc \in \mathcal{C}c∈C and outputs a message m∈Mm \in \mathcal{M}m∈M.

Perfect secrecy

DEFINITION

An encryption scheme (Gen, Enc, Dec) with message space M\mathcal{M}M is perfectly secret if for eevery probability distribution for M, every message m∈Mm \in \mathcal{M}m∈M, and every ciphertext c∈Cc \in \mathcal{C}c∈C for which Pr[C - c] > 0:

Pr[M = m ∣ C = c] = Pr[M = m] Pr[M\ =\ m\ |\ C\ =\ c]\ =\ Pr[M\ =\ m] Pr[M = m ∣ C = c] = Pr[M = m]

An equivalent formulation of perfect secrecy

For every m,m′∈Mm, m' \in \mathcal{M}m,m′∈M, and every c∈Cc \in \mathcal{C}c∈C, we have

Pr[EncK(m) = c] = Pr[EncK(m′) = c] Pr[Enc_K(m)\ =\ c]\ =\ Pr[Enc_K(m')\ =\ c] Pr[EncK​(m) = c] = Pr[EncK​(m′) = c]

PROOF

Pr[C=c ∣ M=m]=Pr[EncK(M)=c ∣ M=m]=Pr[EncK(m)=c ∣ M=m]=Pr[EncK(m)=c] \begin{aligned} Pr[C=c\ |\ M=m] &= Pr[Enc_K(M)=c\ |\ M=m] \\ &= Pr[Enc_K(m)=c\ |\ M=m] \\ &= Pr[Enc_K(m)=c] \end{aligned} Pr[C=c ∣ M=m]​=Pr[EncK​(M)=c ∣ M=m]=Pr[EncK​(m)=c ∣ M=m]=Pr[EncK​(m)=c]​
